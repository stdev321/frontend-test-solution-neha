Phase 1: AI-Powered String Extraction

🤖 Important: This phase uses Claude Code's specialized i18n agents to automatically scan and extract hardcoded strings from React components.

The /i18n-extract Command

⚡ Primary Method: Use the /i18n-extract slash command in Claude Code to orchestrate the complete extraction workflow. 
# In Claude Code, use the slash command:
/i18n-extract frontend/src/pages/FAQPage.jsx

# This command orchestrates 3 agents in sequence:
# 1. i18n-literal-scanner - Finds all hardcoded strings
# 2. i18n-comment-marker - Adds extraction comments
# 3. i18n-comment-executor - Executes the extractions

# The complete workflow:
# → Scans the file for literal strings
# → Adds extraction comments above each literal
# → Processes comments one by one
# → Adds new keys to English JSON files
# → Replaces literals with t() function calls

How the Extraction Agents Work

Agent Purpose Called By
i18n-literal-scanner Scans file and identifies all hardcoded text strings /i18n-extract (step 1)
i18n-comment-marker Adds standardized extraction comments above each literal /i18n-extract (step 2)
i18n-comment-executor Processes comments: extracts to JSON & replaces with t() /i18n-extract (step 3)
Note: These agents are called automatically by /i18n-extract. You don't need to call them individually unless you want manual control.

What Gets Extracted

✅ Extracted:

  • UI text in JSX: <p>Welcome to our app</p>
  • Button labels: <Button>Submit</Button>
  • Placeholder text: placeholder="Enter your email"
  • Error messages: setError("Invalid input")
  • Alert/toast messages: toast.success("Saved successfully!")
  • Dynamic strings: {`Hello, ${userName}!`}

❌ Not Extracted:

  • Console logs: console.log("Debug info")
  • Technical identifiers: className="btn-primary"
  • URLs and API endpoints: fetch("/api/users")
  • Already internationalized: {t('common:welcome')}
  • Comments: // This is a comment

Extraction Comment Format

The agents add standardized comments above literals to be extracted:

// i18n-extract: namespace="pages" key="faq.title"
<h1>Frequently Asked Questions</h1>

// After execution becomes:
<h1>{t('pages:faq.title')}</h1>

Tips for AI Extraction

🔧 Troubleshooting Common Issues

Issue: Smart Quotes Breaking JSON

Error: JSON parse error: Invalid numeric literal

Cause: Copy-pasting from Word/Google Docs introduces curly quotes ("") instead of straight quotes ("")

Fix:

cd frontend
# Fix smart quotes in a file
python3 -c "
import json
with open('public/i18n/locales/zh/aiAccuracyWhitepaper_zh.json', 'r') as f:
    content = f.read()
content = content.replace('"', '\"').replace('"', '\"')
content = content.replace(''', \"'\").replace(''', \"'\")
data = json.loads(content)
with open('public/i18n/locales/zh/aiAccuracyWhitepaper_zh.json', 'w') as f:
    json.dump(data, f, ensure_ascii=False, indent=4)
print('Fixed smart quotes')
"

Issue: Empty Objects Instead of Strings

Error: Validation shows missing keys but sync says "already in sync"

Cause: Values are empty objects {} instead of empty strings ""

Fix:

cd frontend
# Convert empty objects to empty strings
node -e "
const fs = require('fs');
const data = JSON.parse(fs.readFileSync('public/i18n/locales/fil/pages.json', 'utf8'));

function fixEmptyObjects(obj) {
    for (const key in obj) {
        if (typeof obj[key] === 'object' && obj[key] !== null) {
            if (!Array.isArray(obj[key]) && Object.keys(obj[key]).length === 0) {
                obj[key] = '';
            } else if (!Array.isArray(obj[key])) {
                fixEmptyObjects(obj[key]);
            }
        }
    }
    return obj;
}

const fixed = fixEmptyObjects(data);
fs.writeFileSync('public/i18n/locales/fil/pages.json', JSON.stringify(fixed, null, 2) + '\\n');
console.log('Fixed empty objects');
"

Find Empty Values in English

cd frontend
node -e "
const fs = require('fs');
const data = JSON.parse(fs.readFileSync('public/i18n/locales/en/common.json', 'utf8'));
function countEmpty(obj, path = '') {
    let empty = 0;
    for (const key in obj) {
        const fullPath = path ? path + '.' + key : key;
        if (typeof obj[key] === 'object' && obj[key] !== null && !Array.isArray(obj[key])) {
            empty += countEmpty(obj[key], fullPath);
        } else if (obj[key] === '' || (typeof obj[key] === 'string' && obj[key].trim() === '')) {
            empty++;
            if (empty <= 5) console.log('  Empty:', fullPath);
        }
    }
    return empty;
}
const emptyCount = countEmpty(data);
console.log('Total empty values in English common.json:', emptyCount);
"

Updating English Text & Retranslating

When you need to change English text that has already been translated, use this workflow to automatically clear outdated translations and retranslate them.

The Problem

When English text changes, all existing translations become outdated and incorrect. They need to be cleared and retranslated with the new English text.

Example: If "Welcome to our health platform" changes to "Welcome to VirtualMD wellness center", all 29 language translations need updating.

The Solution: Update Workflow

# Step 1: Create backup of current English files
mkdir -p public/i18n/locales/en/old
cp public/i18n/locales/en/*.json public/i18n/locales/en/old/

# Step 2: Edit English files directly in public/i18n/locales/en/
# Make your changes to the English text...

# Step 3: Preview what will be cleared (dry run)
python scripts/i18n-clear-changed.py --dry-run

# Step 4: Clear changed translations in all languages
python scripts/i18n-clear-changed.py

# Step 5: Retranslate the cleared values
python scripts/i18n-translate-all.py all

# Step 6: Clean up backup when done
rm -rf public/i18n/locales/en/old

How It Works

  1. Backup: The en/old/ subfolder stores the original English text for comparison
  2. Compare: The script compares current vs old English to find changed values
  3. Clear: For each changed value, it sets the corresponding translation to empty string in all languages
  4. Retranslate: The translation script fills in all empty values with fresh translations
Why en/old/ subfolder? We use en/old/ instead of en_old to prevent translation scripts from treating it as a language folder.

Script Options

# Process all changed files
python scripts/i18n-clear-changed.py

# Process specific file only
python scripts/i18n-clear-changed.py pages.json

# Preview changes without modifying files
python scripts/i18n-clear-changed.py --dry-run

Important Notes

  • Always backup first: The backup in en/old/ is essential for comparison
  • Edit in place: Make changes directly in the main en/ folder
  • Review with dry-run: Always preview what will be cleared before applying
  • New keys aren't cleared: Only changed values are cleared, new keys are ignored
  • Commit after verification: Review all changes before committing to git
